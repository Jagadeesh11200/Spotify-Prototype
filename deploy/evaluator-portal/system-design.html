System design

Why a query cannot answer what a model can.

The feature needs to understand what a listener emotionally needs right now. That signal is not the same as historical preference, and it cannot be derived only by counting repeat plays.

Module 01 - Thesis

Where fixed rules break

A rule built to find a user's most-played songs will cluster around their historical average. The discovery problem asks for something harder: three options that are meaningful, distinct, and safe enough to start a session with. That requires divergence by design, not just similarity matching.

AI is useful because present-moment state is a pattern across weak signals: time, recency, replay intensity, saves, skips, session context, and mood expression. No single signal is reliable alone.

Module 02 - Data

What the system needs to know

  • Recent listening behavior versus long-term preference.
  • Emotional coordinates such as valence, energy, intensity, warmth, and novelty tolerance.
  • Attachment signals such as replay, return, save, skip, and session position.
  • Research signals from App Store reviews, Play Store reviews, Reddit threads, forums, and interviews.

In this prototype, those production signals are represented by synthetic listening data and deterministic scoring. The MVP proves the interaction layer before integrating a live model and catalog service.

Module 03 - Card selection

How three mood cards get chosen

  1. Score engagement

    Recent tracks are scored from recency, replay, and save-like behavior. Engagement matters more than raw count.

  2. Classify into worlds

    Each track maps into one of fifteen emotional worlds using mood features and style tags.

  3. Weight the center card

    World history can boost the center card, but the boost is capped so one world cannot dominate every session.

  4. Pick distinct side cards

    Side cards are selected for distance from the center and from each other, so the triptych does not feel redundant.

  5. Anchor with memory

    The memory anchor uses known listening context. If the system has no basis for a memory, it should not invent one.

Module 04 - Right Now For You

Generating the discovery section

The selected card carries a mood world, valence, energy, and style profile into Home. The current MVP scores local mock tracks against that profile and surfaces a compact "Right Now For You" section above regular Home content.

In production, this becomes a live candidate generation and re-ranking service: existing Spotify candidates are blended with current intent, novelty control, and evidence from similar emotional sessions.

Module 06 - Why this works

The product bet

The goal is not more recommendations. It is fewer sessions that start from habit alone. Spotify already has enough songs; the missing signal is what the user wants this session to feel like.

  • Built to diverge: three cards should be meaningfully different, not three versions of the same mood.
  • Weight the moment: a recent intentional replay can matter more than months of passive plays.
  • Never block escape: users can skip the intervention, and repeated skips make it compact.
