Evaluator Test Flow

Test whether the MVP moves a listener from repeat behavior to meaningful discovery.

This guide is the suggested walkthrough for evaluating the native Android prototype across the three MVP phases. The goal is not to test every Spotify surface, but to test the discovery intervention and how it changes the listening journey.

Open Mobile Session Open AI Architecture

Primary Question

Does the prototype help a user express what they want to feel, then convert that intent into less repetitive music discovery?

Target User

Habitual streamers who like Spotify, but repeatedly fall back to known artists, old playlists, and familiar tracks because discovery feels noisy or effortful.

Success Signal

The evaluator should see a clear transition from passive replay to active, mood-led exploration with recommendations that feel calibrated to the selected state.

Phase 1

Baseline Spotify Shell

  1. Open the mobile session and observe the familiar Spotify-like home, library, search, player, and navigation surfaces.
  2. Use bottom navigation to move between Home, Search, and Library.
  3. Start a track and confirm the mini player and full player states work.
  4. Like or unlike a track and confirm the library state responds.
Phase 2

Mood Board Discovery Intervention

  1. Enter the Mood Board and review the three-card mood choice pattern.
  2. Pick the mood card that best represents the intended listening state.
  3. Confirm that the app moves into Home with the active mood visible.
  4. Review the "Right Now For You" section and inspect whether the recommendations feel more contextual than a generic playlist.
  5. Play a recommendation and confirm it feeds back into the normal player flow.
Phase 3

Preference Adaptation And Recovery

  1. Exit or skip the Mood Board multiple times to trigger the compact entry behavior.
  2. Return through the mood tile and verify the discovery experience is still available without blocking normal listening.
  3. Use the reset behavior to confirm the prototype can return to the full discovery entry state.
  4. Check that switching between phases does not create stale mood state, broken recommendations, or player regressions.
What To Report

Evaluation Notes

  • Was the mood choice easier than typing a search query or browsing playlists?
  • Did the selected mood create a credible reason to try unfamiliar tracks?
  • Did the experience remain recognizably Spotify, rather than feeling like a separate product?
  • Would this flow reduce repeat listening for the chosen target segment?
